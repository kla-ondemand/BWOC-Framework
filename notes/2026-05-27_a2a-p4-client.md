# 2026-05-27 ‚Äî A2A P4: outbound client

Adds the outbound side of A2A: a BWOC agent can now *initiate* calls to an
external A2A agent ‚Äî the complement of the inbound listener (P1-serve). Fifth
slice of #48 (after P1-core #71, P1-serve #72, P2 #73/#74, P3 #75).

## What changed

- **`bwoc-a2a::client`** (new) ‚Äî `fetch_card(base)` GETs a remote Agent Card
  from its well-known URL; `send_message(endpoint, text, context, message_id)`
  POSTs a JSON-RPC `SendMessage` and returns the `result` (a Task or Message) or
  a typed `ClientError` (incl. surfacing a JSON-RPC error from the peer). Uses
  reqwest with a 30 s timeout.
- **`bwoc-a2a` binary** ‚Äî `fetch-card <url>` and `send <url> <text> [--context]`,
  each wrapping the async client in a one-off current-thread runtime so the
  binary stays a plain `main`.
- **CLI** ‚Äî `bwoc a2a fetch-card` / `bwoc a2a send` forward to the sibling
  binary (the CLI still links no HTTP ‚Äî `bwoc-cli` verified reqwest/tokio-free).

## Decisions

- **reqwest with `rustls-tls`, not native-tls.** No system OpenSSL dependency;
  matches `bwoc-harness`'s reqwest config. `json` feature for (de)serialization.
- **A deserializable response mirror.** The server-side `JsonRpcResponse` is
  serialize-only (`id: &'static str`), so the client parses into a small
  `JsonRpcResponseOwned { result, error }` ‚Äî keeps the wire types one-directional
  rather than forcing `Deserialize` onto the server struct.
- **No signing on outbound (yet).** P4 is plain outbound HTTP to an *external*
  A2A agent, which is not a BWOC trust peer ‚Äî ed25519 signing (HV2-4) is for
  inter-BWOC messages. A signed/authenticated A2A handshake belongs with the
  auth phase, alongside the inbound listener's auth.
- **`messageId` from a nanosecond clock.** A one-shot CLI send needs a unique id
  without pulling a uuid dep; `bwoc-a2a-<unix_nanos>` is sufficient.

## Alternatives considered

- *Make `JsonRpcResponse` `Deserialize` too.* Rejected ‚Äî its `jsonrpc:
  &'static str` field can't round-trip through `Deserialize` cleanly; the small
  owned mirror is simpler than reworking the server type.
- *Stream the response (client-side SSE consumption).* Deferred ‚Äî P4 is the
  unary client; consuming a remote agent's SSE stream can follow if needed
  (would add reqwest's `stream` feature).

## Status / deferred

- Done: `client` module, both binary subcommands + CLI forwarders, 35 `bwoc-a2a`
  tests (5 new client tests via a `wiremock` mock endpoint), live BWOC‚ÜîBWOC
  round-trip (`fetch-card` + `send` against a local `bwoc a2a serve`).
- Deferred: P5 push notifications (the last #48 phase). Outbound signing/auth and
  client-side streaming are future work tied to the auth phase. Dep-quarantine
  unchanged ‚Äî reqwest/tokio/axum stay within the **`bwoc-a2a` crate** (the
  `client` lib module + the binary); `bwoc-cli` and `bwoc-core` link none of
  them (`bwoc-cli` execs the `bwoc-a2a` binary as a subprocess).
