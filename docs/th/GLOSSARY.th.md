# อภิธานศัพท์ — ศัพท์บาลีใน BWOC

เอกสารนี้คือแหล่งค้นหาเร็วสำหรับศัพท์บาลีทุกตัวที่ใช้ใน BWOC แต่ละรายการให้ **ความหมายเชิงวิศวกรรมในหนึ่งบรรทัด** สำหรับการ map อย่างเต็มรูปแบบ (กรอบใดเป็นเจ้าของศัพท์ ใช้ตัดสินอะไร ประกอบกับอะไรได้บ้าง) ดูที่ [`PHILOSOPHY.th.md`](../../modules/agent-template/docs/th/PHILOSOPHY.th.md)

**คำแนะนำการอ่าน:** ศัพท์บาลีเป็น *ป้ายชื่อ* (label) ความหมายเชิงวิศวกรรมคือสิ่งที่สำคัญในการเขียน code และ review ไม่จำเป็นต้องมีความรู้ทางศาสนาเพื่อใช้ BWOC

---

## วิธีใช้

- **เจอศัพท์ในเอกสารหรือ commit?** หาในตารางเรียงตามอักษรด้านล่าง
- **อยากรู้ว่าศัพท์อยู่ในกรอบไหน?** คอลัมน์ "ความหมายเชิงวิศวกรรม" บอกชื่อกลุ่ม
- **อยากดูการอธิบายเต็ม?** กดลิงก์ไปยัง `PHILOSOPHY.th.md` ที่ท้ายเอกสาร

---

## ดัชนีเรียงตามอักษร

> ลำดับตาม Pali transliteration (อักษรโรมันกำกับเสียง) — ใช้เป็น key ค้นหาที่เป็นกลางต่อภาษา

| บาลี | คำแปล | ความหมายเชิงวิศวกรรม |
|---|---|---|
| **Acinteyya 4** (อจินไตย 4) | สิ่งที่ไม่ควรคิด | สิ่งที่จงใจไม่ model — ขอบเขตของการคาดเดา |
| **Adinnādānā veramaṇī** (อทินนาทาน) | งดเอาของที่ไม่ได้ให้ | CoC: เคารพ attribution, license, IP ไม่ลอกผลงาน |
| **Anattā** (อนัตตา) | ไม่ใช่ตัวตน | ปล่อยสถานะเก่า ล้าง worktree, branch, memory ไม่ยึดมั่น |
| **Anicca** (อนิจจัง) | ความไม่เที่ยง | ทุกอย่างเปลี่ยนแปลง memory และ cache ต้องถูกตัดและประทับเวลา |
| **Aparihāniya-dhamma 7** (อปริหานิยธรรม 7) | หลัก 7 ประการแห่งความไม่เสื่อม | Fleet governance — สิ่งที่ทำให้ระบบหลาย agent ไม่เสื่อม |
| **Ariya-dhana 7** (อริยทรัพย์ 7) | ทรัพย์อันประเสริฐ 7 | ระดับความเจริญของ capability (L1 → L7) |
| **Ariyasacca 4** (อริยสัจ 4) | ความจริงอันประเสริฐ 4 | แกนการแก้ปัญหา ทุก task: ทุกข์ → สมุทัย → นิโรธ → มรรค |
| **Attanutata** (อัตตัญญุตา) | รู้ตน | ประกาศความสามารถ — agent บอกล่วงหน้าว่าทำอะไรได้และทำอะไรไม่ได้ |
| **Bhāvanā 4** (ภาวนา 4) | การเจริญ 4 | ระยะวงจรชีวิตของ agent (เติบโต → เจริญ → สอน → ปล่อย) |
| **Bhava-taṇhā** (ภวตัณหา) | ความอยากเป็น/อยู่ | ประเภทภัยคุกคาม: การคงอยู่ ยกระดับสิทธิ |
| **Brahmavihāra 4** (พรหมวิหาร 4) | ธรรมเครื่องอยู่ของพรหม 4 | ทัศนคติด้าน UX/error: เมตตา · กรุณา · มุทิตา · อุเบกขา |
| **Dukkha** (ทุกข์) | ทุกข์ / ปัญหา | (1) สัจจะข้อที่ 1 — นิยามปัญหาที่จับต้องได้; (2) หนึ่งในไตรลักษณ์ |
| **Iddhipāda 4** (อิทธิบาท 4) | ฐานแห่งความสำเร็จ 4 | เครื่องยนต์ของงาน — ฉันทะ วิริยะ จิตตะ วิมังสา |
| **Jāti** (ชาติ) | การเกิด | คำพ้องระดับหยาบของ [Uppāda] (เป็นทางเลือก) |
| **Kamma 3** (กรรม 3) | ทวารกรรม 3 | Audit logging — กาย วาจา ใจ → commit, message, plan |
| **Kalyāṇamitta 7** (กัลยาณมิตร 7) | ลักษณะกัลยาณมิตร 7 | คะแนนความเชื่อใจระหว่าง agent |
| **Kāma-taṇhā** (กามตัณหา) | ความอยากในสิ่งเร้า | ประเภทภัยคุกคาม: การชักจูง (prompt injection, social engineering) |
| **Kāmesumicchācārā veramaṇī** (กาเมสุมิจฉาจาร) | งดประพฤติผิดในกาม | CoC: เคารพขอบเขต ไม่มีพฤติกรรมเชิงเพศที่ไม่พึงประสงค์ใน project channels |
| **Karuṇā** (กรุณา) | ความเอ็นดู | UX: เสนอทางแก้ ไม่ใช่แค่รายงานความผิดพลาด |
| **Khandha 5** (ขันธ์ 5) | ขันธ์ 5 | สถาปัตยกรรม — รูป · เวทนา · สัญญา · สังขาร · วิญญาณ → file/IO/memory/logic/runtime |
| **Magga** (มรรค) | ทาง | (1) สัจจะข้อที่ 4 — แผน; (2) ย่อจาก Magga 8 |
| **Magga 8** (มรรค 8) | มรรคมีองค์ 8 | Functional requirements — 8 เสาใน SRS |
| **Maraṇa** (มรณะ) | ความตาย | คำพ้องระดับหยาบของ [Vaya] (เป็นทางเลือก) |
| **Mattaññutā** (มัตตัญญุตา) | รู้ประมาณ | วินัยความพอดี — `MEMORY.md` ≤ 200 บรรทัด; สเปกเล็กกว่าชนะ |
| **Mettā** (เมตตา) | ความรัก | UX: น้ำเสียงเป็นมิตร ตรงไปตรงมา |
| **Muditā** (มุทิตา) | ความยินดี | UX: ยอมรับเมื่อผู้อื่นถูก ต้อนรับผู้ contribute ใหม่ |
| **Musāvādā veramaṇī** (มุสาวาท) | งดพูดเท็จ | CoC: ไม่ปลอมตัว ไม่ปลอม results ไม่เขียน commit message ที่ทำให้เข้าใจผิด |
| **Nirodha** (นิโรธ) | ความดับ | (1) สัจจะข้อที่ 3 — สภาวะสำเร็จที่วัดได้; (2) การกระทำในระยะ *วยะ* |
| **Padhāna 4** (ปธาน 4) | ความเพียร 4 | วินัยความเพียร — สังวร ปหาน ภาวนา อนุรักขนา |
| **Paññā 3** (ปัญญา 3) | รากของปัญญา 3 | Self-improvement — สุตมยปัญญา จินตามยปัญญา ภาวนามยปัญญา |
| **Pāṇātipātā veramaṇī** (ปาณาติบาต) | งดเบียดเบียน | CoC: ไม่ harass, ขู่, doxx, หรือ hate speech |
| **Paṭiccasamuppāda** (ปฏิจจสมุปบาท) | ปฏิจจสมุปบาท | Failure analysis — สาวเหตุปัจจัยกลับไป ปัญหาที่เห็นมักไม่ใช่รากเหตุ |
| **Sammā-ājīva** (สัมมาอาชีวะ) | เลี้ยงชีพชอบ | ความเชื่อใจและความเป็นกลาง — ไม่ผูกขาด vendor |
| **Sammā-diṭṭhi** (สัมมาทิฏฐิ) | เห็นชอบ | Persona, identity |
| **Sammā-kammanta** (สัมมากัมมันตะ) | ทำการชอบ | วินัย worktree, commits |
| **Sammā-samādhi** (สัมมาสมาธิ) | ตั้งใจชอบ | Focus, ขอบเขต session |

| **Sammā-sati** (สัมมาสติ) | ระลึกชอบ | ระบบความจำ |
| **Sammā-vācā** (สัมมาวาจา) | พูดชอบ | การสื่อสารระหว่าง agent |
| **Sammā-vāyāma** (สัมมาวายามะ) | เพียรชอบ | Verification gates (lint, format, test, regression, build) |

| **Saṅkhata** (สังขตะ) | สิ่งปรุงแต่ง | สิ่งที่เกิดขึ้นและดับไป รากของวงรอบ (อุปฺปาท · ฐิติ · วยะ) |
| **Sappurisadhamma 7** (สัปปุริสธรรม 7) | ธรรมของสัตบุรุษ 7 | Context sensing — สถานการณ์ ผู้ฟัง เวลา ฯลฯ |
| **Sāraṇīyadhamma 6** (สาราณียธรรม 6) | ธรรมเป็นเหตุให้ระลึกถึงกัน 6 | โปรโตคอลการประสานงานระหว่าง agent |
| **Satipaṭṭhāna 4** (สติปัฏฐาน 4) | สติปัฏฐาน 4 | Observability — กาย เวทนา จิต ธรรม → metrics, logs, traces, state |

| **Sīla 5** (ศีล 5) | ศีล 5 | การกระทำที่ห้ามพื้นฐาน — ไม่เบียดเบียน ไม่ลัก ไม่ผิดในกาม ไม่พูดเท็จ ไม่ประมาท |

| **Surāmerayamajjapamādaṭṭhānā veramaṇī** (สุราเมรยมัชชปมาทัฏฐาน) | งดเครื่องเสพย์ติดที่ทำให้ประมาท | CoC: ไม่ contribute ใต้สภาพที่ตัดสินใจไม่ได้ ไม่ commit แบบประมาท |
| **Taṇhā 3** (ตัณหา 3) | ตัณหา 3 | ประเภทภัยคุกคาม — กาม · ภวะ · วิภวะ (สิ่งเร้า · คงอยู่ · ทำลาย) |
| **Ṭhiti** (ฐิติ) | ตั้งอยู่พร้อมแปรไป | ระยะที่ 2 ของวงรอบ — agent ดำเนินงาน สถานะแปรไปภายใต้วินัย |
| **Tilakkhaṇa** (ไตรลักษณ์) | ไตรลักษณ์ | อนิจจัง · ทุกขัง · อนัตตา — การออกแบบต้องสอดคล้อง |
| **Upekkhā** (อุเบกขา) | วางใจเป็นกลาง | UX: สงบเมื่อขัดเคือง ไม่ตอบโต้ |
