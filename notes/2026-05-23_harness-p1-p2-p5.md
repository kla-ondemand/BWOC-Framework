# 2026-05-23 — bwoc-harness: P1 + P2 + P5 (issue #1 milestone)

Built the BWOC-native agentic harness (`crates/bwoc-harness`) per [`2026-05-23_ollama-agentic-harness-design.md`](2026-05-23_ollama-agentic-harness-design.md), through three phases, on branch `feat/harness-p1` (off `main` @ db1a466). Orchestrated by agent-oracle via controlled sub-agents (the tmux Pi/Codx agents were not reliably drivable); each phase was Oracle-reviewed + gate-verified before stacking the next. **Not merged to main** — held on the branch for the operator's review (security-sensitive, identity-expanding).

## What changed

- **P1 (`7af2cd9`)** — new crate `crates/bwoc-harness` (added to workspace members). `provider/` = OpenAI-compatible `/v1/chat/completions` client behind a mockable `ProviderClient` trait (`stream=false` + SSE `stream=true`; model-existence validation → clear 404 error). `agent_loop.rs` = the turn loop (call → accumulate tool_calls → dispatch → append results → repeat → stop on final/max-iter/cancel). `tools/` = registry + `read_file`/`write_file`/`list_dir`/`run_command` with worktree path confinement. `main.rs` = clap CLI loading AGENTS.md/CLAUDE.md as system prompt. Stubs for telemetry/policy/sandbox/queue/eval.
- **P2 (`43bbc31`)** — safety pipeline, dispatch order **guardrails → permission → sandbox → execute**. `policy/guardrails.rs` = hard, non-overridable floor mapped to Sīla 5 + Taṇhā 3 (rm -rf root, secret writes, identity spoof, gate bypass, sudo). `policy/permission.rs` = per-tool/pattern `allow|ask|deny` from manifest + `.bwoc/harness-policy.toml`; `ask` on non-TTY → `deny` (fail-safe). `sandbox.rs` = path confinement (canonicalize existing / lexical for new), env scrub, arg-level command scan, OS-sandbox left as a `OsSandbox` trait stub (no-op). Denials are fed back to the model as tool results, never panics.
- **P5 (`f947c68`)** — `Backend::Ollama` added; `bwoc spawn --backend ollama` execs the `bwoc-harness` **binary** (resolved sibling-of-`bwoc` → `CARGO_BIN_EXE` → `$PATH`), not an external CLI. `OLLAMA.md → AGENTS.md` template symlink. `doctor` reports harness-binary + endpoint reachability (TCP, no HTTP dep). check/status/banner/chat/help/new updated. `check-agent-neutrality.sh` passes.

## Decisions

- **Heavy deps quarantined in `bwoc-harness` only** — bwoc-cli/bwoc-agent/bwoc-core gain no `tokio`/`reqwest`/`async-trait`/`hyper` (verified each phase). The harness is exec'd as a subprocess, never linked — the zero-dep-orchestrator promise holds for the default path.
- **Guardrails before permission** — resolved a minor inconsistency in the design note (§Architecture line said permission→guardrails; the component table said guardrails are non-overridable). Hard floor runs first so permission can never grant what guardrails forbid.
- **OS-level sandbox deferred to a stub trait** — v1 is worktree+allowlist only (the design's open question, resolved for v1).
- **Held on branch, not merged to main** — operator reviews before landing (per "พี่รอตรวจ").
